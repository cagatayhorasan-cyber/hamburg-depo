👥1. Rol sistemi ve kullanıcı yönetimi1. Rollen und Benutzerverwaltung

Sistemde dört rol vardır: admin, staff (personel), operator, customer. Her birinin erişim seviyesi farklıdır.

Das System kennt vier Rollen: admin, staff (Personal), operator, customer (Kunde). Jede Rolle hat unterschiedliche Zugriffsrechte.

Rol yetkileri

Rollenberechtigungen

  • Admin: Her şeye tam erişim — stok, fiyat, kullanıcı, güvenlik, finans.
  • Staff: Satış/teklif, stok görüntüleme, müşteri siparişi; fiyat değiştirme ve kullanıcı yönetimi yok.
  • Operator: Sadece stok giriş/çıkış kayıtları.
  • Customer: Kendi hesabı — katalog, kendi siparişleri, kendi projeleri.
  • Admin: Vollzugriff — Bestand, Preise, Benutzer, Sicherheit, Finanzen.
  • Staff: Verkauf/Angebot, Bestand einsehen, Kundenbestellungen; keine Preisänderungen, keine Benutzerverwaltung.
  • Operator: Nur Wareneingang/-ausgang erfassen.
  • Customer: Eigenes Konto — Katalog, eigene Bestellungen, eigene Projekte.

Yeni kullanıcı oluşturma

Neuen Benutzer anlegen

  1. Sol menüden Kullanıcılar sekmesine geç.
  2. Sağ üstte "Yeni Kullanıcı" butonu.
  3. Rol seç (Staff / Operator / Customer), adı + kullanıcı adı + parola + email gir.
  4. Müşteri için email doğrulaması otomatik gönderilir.
  1. Im linken Menü den Tab Benutzer öffnen.
  2. Oben rechts "Neuer Benutzer".
  3. Rolle wählen (Staff / Operator / Customer), Name + Benutzername + Passwort + E-Mail eingeben.
  4. Für Kunden wird die E-Mail-Bestätigung automatisch versendet.
Dikkat: Admin kullanıcı silme işlemi geri alınamaz. Silme yerine rolünü düşürmeyi tercih et.
Achtung: Das Löschen von Admin-Benutzern ist nicht rückgängig zu machen. Besser die Rolle herabstufen.

📦2. Stok yönetimi2. Bestandsverwaltung

Yeni ürün ekleme

Neuen Artikel hinzufügen

  1. Stok sekmesinde sağdaki "Yeni Ürün + İlk Stok Girişi" formunu doldur.
  2. Zorunlu: Ad, Birim, Kritik Stok, Stok Kodu (DRC-XXXXX formatında önerilir).
  3. Opsiyonel: Marka, Kategori, Alış Fiyatı, Satış Fiyatı, Liste Fiyatı, Açıklama.
  4. İlk stok miktarını da gir — tek adımda ürün + giriş hareketi oluşturulur.
  1. Im Tab Bestand das Formular "Neuer Artikel + Erste Einlagerung" ausfüllen.
  2. Pflichtfelder: Name, Einheit, Mindestbestand, Lagercode (Format DRC-XXXXX empfohlen).
  3. Optional: Marke, Kategorie, Einkaufspreis, Verkaufspreis, Listenpreis, Beschreibung.
  4. Anfangsmenge angeben — Artikel und erste Bewegung werden in einem Schritt erstellt.

Stok giriş/çıkış hareketi

Wareneingang/-ausgang erfassen

Stok sekmesinde "Stok Giriş / Çıkış" formunu kullan. Giriş → stok artar, çıkış → azalır. Her hareket tarihli loglanır.

Im Tab Bestand das Formular "Wareneingang / -ausgang" verwenden. Eingang = Bestand steigt, Ausgang = sinkt. Jede Bewegung wird mit Datum protokolliert.

Kritik stok uyarıları

Mindestbestand-Warnungen

Her ürün için "Kritik Stok" değeri tanımla. Mevcut stok bu değerin altına düşünce kart kırmızı olur ve listede işaretlenir. Admin dashboard'a özet düşer.

Für jeden Artikel einen "Mindestbestand" definieren. Fällt der Bestand darunter, wird die Karte rot markiert. Admin-Dashboard zeigt eine Übersicht.

📄3. Teklif, satış ve sipariş3. Angebot, Verkauf und Bestellung

Teklif oluşturma

Angebot erstellen

  1. Teklif sekmesine geç.
  2. Müşteri adı (serbest metin veya kayıtlı müşteri dropdown'undan seç).
  3. Başlık, tarih, açıklama gir.
  4. "Ürün Ekle" ile stoktan kalemler seç veya elle ürün adı yaz.
  5. İskonto %, KDV (%19 veya %0 ihracat) ayarla.
  6. Dil seç (TR veya DE) → PDF buna göre üretilir.
  7. "Teklif Oluştur" → PDF indirmeye hazır.
  1. Tab Angebot öffnen.
  2. Kundenname (frei oder aus Dropdown).
  3. Titel, Datum, Notiz eingeben.
  4. "Artikel hinzufügen" — Lagerartikel wählen oder Artikelname manuell eintragen.
  5. Rabatt %, MwSt (19 % oder 0 % Export) setzen.
  6. Sprache wählen (TR/DE) → PDF wird entsprechend erstellt.
  7. "Angebot erstellen" → PDF steht zum Download bereit.

Satış (POS)

Verkauf (POS)

Satış tamamlandığında stok otomatik düşer. Ödeme türü (Nakit/Banka/Kart) ve tahsil edilen tutar girilir. Kısmi ödeme varsa "Açık hesap" olarak işaretlenir.

Beim Verkaufsabschluss wird der Bestand automatisch reduziert. Zahlungsart (Bar/Bank/Karte) und eingenommener Betrag erfasst. Teilzahlung wird als "Offene Rechnung" markiert.

Sipariş yönetimi

Bestellungsverwaltung

Siparişler sekmesinde tüm müşteri siparişlerini görürsün. Durum: Bekliyor → Onaylandı → Hazırlanıyor → Tamamlandı. Onay anında stok düşer. Durumu "İptal" yaparsan stok geri gelir (stock_deducted_at null'sa).

Im Tab Bestellungen alle Kundenbestellungen. Status: Ausstehend → Bestätigt → In Bearbeitung → Abgeschlossen. Bei Bestätigung wird Bestand reduziert. "Storniert" setzt Bestand zurück (wenn stock_deducted_at = NULL).

🧊4. Projeler (ColdRoomPro)4. Projekte (ColdRoomPro)

Projeler sekmesi soğuk oda projelerini barındırır — hesaplama, BOM (malzeme listesi), teklife/siparişe çevirme.

Der Tab Projekte enthält Kühlraumprojekte — Berechnung, BOM (Stückliste), Umwandlung in Angebot/Bestellung.

Yeni proje oluşturma

Neues Projekt erstellen

  1. Manuel: "+ Yeni Proje" → başlık, tip, müşteri, açıklama.
  2. ColdRoomPro ile: "🧊 ColdRoomPro ile Hesapla" → iframe açılır, hesaplama sonunda otomatik proje oluşur.
  1. Manuell: "+ Neues Projekt" → Titel, Typ, Kunde, Notiz.
  2. Mit ColdRoomPro: "🧊 Mit ColdRoomPro berechnen" → iframe öffnet, nach Berechnung wird das Projekt automatisch erstellt.

Projeden teklif/sipariş üretme

Angebot/Bestellung aus Projekt

  • Teklife Çevir: BOM → quote_items olarak kopyalanır, PDF üretilebilir.
  • Siparişe Çevir: Müşteri atanmışsa direkt order oluşturulur.
  • In Angebot umwandeln: BOM wird zu quote_items kopiert, PDF erstellbar.
  • In Bestellung umwandeln: Wenn Kunde zugeordnet, wird direkt eine Bestellung erstellt.

💰5. Finans: Gelir, gider, kasa5. Finanzen: Einnahmen, Ausgaben, Kasse

  • Gelir: Satışlardan otomatik kayıt + manuel gelir eklenebilir.
  • Gider: Tedarikçi ödemeleri, operasyonel giderler — kategorize edilmeli.
  • Kasa: Nakit/Banka/Kart bakiyeleri. Gün sonu kapanış raporu.
  • Raporlar: Aylık net, müşteri bazlı alacak, en çok satan ürünler.
  • Einnahmen: Automatisch aus Verkäufen + manuelle Einträge.
  • Ausgaben: Lieferantenzahlungen, Betriebskosten — kategorisieren.
  • Kasse: Bar-/Bank-/Kartensalden. Tagesabschluss.
  • Berichte: Monatsnettos, Forderungen pro Kunde, Top-Artikel.

🌐6. Toplu DE çeviri (yeni)6. Bulk DE-Übersetzung (neu)

Tüm ürünlerin adını ve notunu TR→DE otomatik çevirme aracı. Sadece name_de / notes_de boş olan kayıtlara dokunur.

Tool zur automatischen TR→DE-Übersetzung aller Artikelnamen und Notizen. Nur leere name_de / notes_de Felder werden gefüllt.

  1. Stok sekmesine geç.
  2. "Stokta Olan Ürünler" başlığının yanındaki 🌐 Toplu DE Çeviri butonuna bas.
  3. Onay diyaloğu gelir, kabul et.
  4. 500'lü batch'ler halinde otomatik ilerler, progress bar gösterir.
  5. Bitince "X güncellendi, Y atlandı" özeti.
  1. Tab Bestand öffnen.
  2. Neben "Artikel am Lager" den Button 🌐 Bulk DE-Übersetzung klicken.
  3. Im Bestätigungsdialog fortfahren.
  4. Läuft automatisch in 500er-Batches mit Fortschrittsanzeige.
  5. Am Ende Zusammenfassung "X aktualisiert, Y übersprungen".
Çeviri kural tabanlı (teknik sözlük). Yanlış çıkan kayıtları manuel olarak ürün kartından düzeltebilirsin.
Regelbasierte Übersetzung (Fachwörterbuch). Falsche Einträge können manuell in der Artikelkarte korrigiert werden.

🔒7. Güvenlik ve olay kayıtları7. Sicherheit und Ereignisprotokoll

Güvenlik sekmesinde tüm sistem olayları: girişler, başarısız denemeler, rate-limit bloklar, admin işlemleri.

Im Tab Sicherheit alle Systemereignisse: Logins, Fehlversuche, Rate-Limit-Blocks, Admin-Aktionen.

  • Brute-force bloklarını manuel kaldırabilirsin.
  • Şüpheli IP'leri kalıcı banlayabilirsin (yakında).
  • Olay severity: info / warn / error / critical.
  • Brute-Force-Blocks manuell aufheben möglich.
  • Verdächtige IPs dauerhaft sperren (demnächst).
  • Ereignis-Severity: info / warn / error / critical.

💬8. Mesajlar ve müşteri destek8. Nachrichten und Kundensupport

Müşteriler portal içinden admin'e mesaj gönderebilir (destek talebi, teknik soru). Admin panelinden yanıtlanır.

Kunden können aus dem Portal Nachrichten an den Admin senden (Supportanfrage, technische Frage). Beantwortung über Admin-Panel.

⚙️9. Sistem ayarları9. Systemeinstellungen

  • Portal dili TR/DE — her kullanıcı kendi seçimini kaydeder.
  • PDF logo ve firma bilgisi (Impressum) — server tarafından.
  • KDV oranı (varsayılan %19), ihracat 0 %.
  • Kritik stok eşikleri ürün bazında.
  • Portalsprache TR/DE — pro Benutzer gespeichert.
  • PDF-Logo und Firmendaten (Impressum) — serverseitig.
  • MwSt-Satz (Standard 19 %), Export 0 %.
  • Mindestbestand pro Artikel.
